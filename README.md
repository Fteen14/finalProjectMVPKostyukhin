# Password Manager AI MVP

Веб-приложение для генерации, хранения и анализа паролей с использованием локальной логики оценки сложности и AI-анализа через YandexGPT API.

## Возможности

- авторизация пользователя
- генерация уникальных паролей по заданным требованиям
- добавление собственного пароля
- проверка надёжности пароля по шкале от 0 до 10
- цветная шкала безопасности
- проверка на распространённые слабые пароли
- защита от повторного сохранения одного и того же пароля
- поиск по сохранённым записям
- удаление записей
- показ и скрытие сохранённого пароля
- AI-анализ пароля через YandexGPT
- AI-улучшение пароля через YandexGPT

## Используемые технологии

- Python
- Flask
- HTML / CSS
- YandexGPT API
- JSON для хранения данных
- python-dotenv
- requests

## Структура проекта

- `app.py` — основной Flask-сервер
- `ai.py` — работа с YandexGPT API
- `utils.py` — логика генерации, оценки и хранения паролей
- `config.py` — загрузка переменных окружения
- `templates/login.html` — страница входа
- `templates/app.html` — основная страница приложения
- `static/style.css` — стили интерфейса
- `common_passwords.txt` — список слабых паролей
- `passwords.json` — локальное хранилище записей
- `.env` — приватные настройки проекта

## Установка и запуск

### 1. Клонировать проект
```bash
git clone <ссылка_на_репозиторий>
cd <имя_папки_проекта>
```

### 2. Создать виртуальное окружение
```bash
python -m venv venv
```

### 3. Активировать виртуальное окружение

#### Windows (Git Bash)
```bash
source venv/Scripts/activate
```

#### Windows (PowerShell)
```bash
venv\Scripts\Activate.ps1
```

### 4. Установить зависимости
```bash
pip install -r requirements.txt
```

### 5. Создать файл `.env`
Создай в корне проекта файл `.env` и добавь в него:

```env
APP_USER=user
APP_PASSWORD=password
SECRET_KEY=super_secret_key_change_me

YANDEX_API_KEY=ВАШ_API_КЛЮЧ
YANDEX_FOLDER_ID=ВАШ_FOLDER_ID

DATA_FILE=passwords.json
COMMON_PASSWORDS_FILE=common_passwords.txt
```

## Запуск проекта

```bash
python app.py
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:5000
```

## Данные для входа

Логин и пароль задаются в `.env`.

Пример:

- логин: `user`
- пароль: `password`

## Использование YandexGPT

В проекте AI используется для двух задач:

- анализа введённого пароля
- генерации улучшенной версии пароля

Если YandexGPT временно недоступен или неправильно настроены ключи, приложение всё равно продолжит работать за счёт локальной оценки сложности.

## Безопасность

- приватные ключи и настройки хранятся в `.env`
- `.env` исключён из Git через `.gitignore`
- пароли дополнительно хранятся в виде SHA-256 хеша
- для демонстрации в MVP пароль также хранится в открытом виде для отображения в интерфейсе
